# Security Policy

感谢你帮助 WilberFlow 保持安全。

这个项目是一个本地运行的地震波形工作流工具，会接触到一些相对敏感的信息，例如：

- QQ 邮箱账号
- QQ IMAP 授权码
- GitHub Token
- 本地文件路径
- Wilber 成功邮件中的下载链接

因此，如果你发现的是安全相关问题，请尽量不要把完整细节直接公开贴到 issue 中。

## 适合私下报告的问题

下面这些情况更适合私下报告：

- 仓库中误提交了密钥、授权码、Token、Cookie
- 配置导出、日志或错误信息会泄露敏感字段
- 本地文件写入、路径处理或下载逻辑存在明显安全缺陷
- 能够导致凭据泄露、任意覆盖文件或意外暴露私有数据的问题

## 不属于安全漏洞的情况

下面这些通常更适合直接开公开 issue：

- 功能建议
- 使用问题
- UI 体验问题
- 文档不清楚
- 一般性运行报错

## 如何报告

如果问题中包含敏感内容，请不要把原始授权码、Token、Cookie、完整邮箱地址或完整下载链接直接公开贴出。

你可以：

1. 先在本地撤销或更换相关凭据
2. 再整理一份去敏感后的问题描述
3. 通过 GitHub 账号主页的联系方式或仓库维护者可用的私下渠道联系

如果暂时没有单独的安全邮箱，至少请先：

- 立即撤销已泄露的凭据
- 不要在公开 issue 中粘贴完整 secret
- 只提供最小必要信息帮助复现

## 维护者处理建议

如果发现仓库中已经出现 secret 泄露，建议优先执行：

1. 立即撤销对应凭据
2. 从仓库和 release 资产中移除敏感内容
3. 评估是否需要重写 Git 历史
4. 在必要时补充公开说明，但不要再次暴露 secret 本体

## 支持版本

目前安全修复优先针对：

- `main`
- 最新 release

较旧版本可能不会单独维护。
