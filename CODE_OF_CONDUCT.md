# Code of Conduct

## Our Goal

WilberFlow 希望成为一个让不同背景用户都能安心使用和协作的项目。

无论你是：

- 第一次接触 Wilber 自动化的新用户
- 想贡献代码、文档或测试的协作者
- 主要关注地震学处理流程而不是软件工程细节的研究者

都应该能够在这里获得尊重、清晰沟通和建设性的反馈。

## Expected Behavior

参与本项目时，我们希望大家：

- 以尊重、耐心和合作的方式交流
- 讨论问题本身，而不是攻击个人
- 对新手问题保持友好，不羞辱基础提问
- 在给反馈时尽量具体、诚实、可执行
- 接受技术分歧存在，但避免情绪化升级

## Unacceptable Behavior

下面这些行为不适合出现在本项目中：

- 人身攻击、羞辱、讽刺、恶意挑衅
- 基于背景、身份、经验水平等进行歧视或排斥
- 故意公开他人的敏感信息
- 持续性骚扰、威胁或破坏讨论氛围
- 明知会造成混乱，仍反复发布无关、恶意或误导性内容

## Scope

本行为准则适用于：

- GitHub issues
- Pull requests
- Discussions 或其他公开协作空间
- 与本项目相关的沟通场景

## Reporting

如果你遇到了不合适的行为：

1. 尽量保留必要证据
2. 不要在情绪升级的现场继续拉扯
3. 通过仓库维护者可用的联系渠道反馈

如果问题同时涉及敏感信息泄露，请优先参考：

- [SECURITY.md](SECURITY.md)

## Maintainer Responsibility

维护者可以根据情况采取以下措施：

- 提醒并要求调整表达方式
- 隐藏、编辑或删除不当内容
- 关闭失控讨论
- 拒绝继续与恶意行为协作

处理标准会尽量以“保护协作环境”为优先，而不是鼓励对抗。

## Notes

这个行为准则的目标不是增加形式负担，而是让这个仓库在公开使用时，仍然保持清晰、友好和可协作。
